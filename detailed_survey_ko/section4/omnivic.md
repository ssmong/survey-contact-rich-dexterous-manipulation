### 4.2 OmniVIC

**전체 제목:** OmniVIC: Omnidirectional Variable Impedance Control via Vision-Language Model with Self-Improving RAG

**저자:** Luigi Penco, Niko Suenderhauf, Fares Abu-Dakka, et al. (IIT Genoa / Georgia Tech)

**학회/연도:** arXiv 프리프린트, 2025

**K/D 결정 방법:** VLM (GPT-4o-mini) + 자기 개선 RAG (Retrieval-Augmented Generation). 이전 작업 실행과 관련 임피던스 파라미터 및 결과의 데이터베이스 유지. 새 작업 시 VLM이 RAG로 관련 과거 경험을 검색하여 K/D 값을 추론하고 제안. 실행 후 성공/실패 피드백을 데이터베이스에 저장하여 시간이 지남에 따라 자기 개선.

**출력:** 전방향 임피던스 제어를 위한 전체 6D 직교 강성 행렬 K 및 감쇠 행렬 D. 파라미터는 방향 인식으로 서로 다른 직교 축에서 다른 유연성 허용.

**로봇 플랫폼:** Franka Emika Panda + 6축 F/T 센서 + 병렬 조 그리퍼. 정교한 핸드 없음.

**작업:** 힘 조절을 포함한 표면 닦기, 페그인홀 삽입, 물체 연마, 유연한 전달 등 접촉이 풍부한 조작. 방향 의존 유연성이 필요한 작업(예: 삽입 축에서 강성, 횡방향에서 유연).

**핵심 방법론:** OmniVIC는 자기 개선 RAG 루프로 기본 VLM 기반 임피던스 생성을 확장. VLM이 작업 설명, 현재 시각 관측, 임피던스 파라미터와 결과가 포함된 유사 과거 작업의 검색된 예시를 수신. 방향별 K/D 값 생성. F/T 센서가 실행 품질 평가와 RAG 데이터베이스 업데이트에 사용되는 힘 피드백 제공. 반복 시행에 걸쳐 그래디언트 기반 훈련 없이 임피던스 파라미터 선택 개선.

**주요 기여:**
- 실행 결과로부터 그래디언트 업데이트 없이 학습할 수 있는 임피던스 파라미터 선택을 위한 자기 개선 RAG 도입
- 스칼라 강성을 넘어 VLM 추론에서의 전방향(축별) 임피던스 생성 시연
- RAG 검색이 반복 시행에 걸쳐 순수 제로샷 VLM 프롬프팅 대비 K/D 선택 정확도를 향상시킴을 보여줌

**한계점:**
- 정교한 핸드 없음 -- Franka Panda 그리퍼 전용
- 개선 루프에 다수의 실세계 시행 필요
- 상용 VLM API(GPT-4o-mini)에 의존
- 접촉 중 연속적 임피던스 적응 미대응(작업 단계별 파라미터 설정)
- VLM 추론 지연(~100ms-1s)이 임피던스 업데이트를 ~1-10 Hz로 제한

**결과:** 자기 개선 RAG가 모든 평가 작업에서 순수 제로샷 VLM 임피던스 생성을 능가. 여러 반복 후 전문가 조정 값에 비견되는 임피던스 파라미터로 수렴. F/T 센서 피드백이 고정 임피던스 기준선 대비 최대 힘 감소 및 작업 성공률 향상 확인.

## 추론 / 배포
- **추론 지연 시간:** 명시적 벤치마크 없음. GPT-4o-mini API 쿼리당 ~100ms-1s + FAISS 검색 오버헤드(~10ms 이하).
- **배포 하드웨어:** 6축 F/T 센서가 장착된 Franka Emika Panda.
- **실시간 가능 여부:** 임피던스 적응에 대해 아니오. 저수준 임피던스 컨트롤러는 Franka에서 실시간으로 실행되나, VLM 기반 업데이트는 API 호출 지연으로 ~1-10 Hz 제한.

## 데이터셋 / 데이터 수집
- **사용 데이터셋:** LIBERO 벤치마크(20개 작업)와 맞춤 RAG 지식 베이스.
- **데이터 규모:** RAG 데이터베이스: 200개 레코드(10개 작업에 걸쳐 작업-단계 쌍당 20개 레코드; 최대 용량 20,000). 평가: 10개 쿼리셋 작업에서 방법당 작업당 50개 에피소드.
