# 10.11 DexPilot

- **全称：** DexPilot: Vision Based Teleoperation of Dexterous Robotic Hand-Arm System
- **作者：** Ankur Handa, Karl Van Wyk, Wei Yang, Jacky Liang, Yu-Wei Chao, Qian Wan, Stan Birchfield, Nathan Ratliff, Dieter Fox (NVIDIA)
- **发表venue/年份：** ICRA 2020 (arXiv 1910.03135, 2019年10月)
- **arXiv：** https://arxiv.org/abs/1910.03135
- **输入模态：** 基于视觉（RGB相机观察裸手，无手套或标记物）
- **目标手：** Allegro Hand（16自由度）+ Kuka IIWA机械臂（7自由度），共23个驱动自由度
- **力反馈：** 无
- **成本：** 低（单目RGB相机）

## 核心方法/设计

DexPilot通过单目RGB相机观察操作者的裸手来实现灵巧机器人手臂系统的遥操作。系统使用基于视觉的手部姿态估计器检测手部关键点（指尖、指关节、手腕），然后求解优化问题将检测到的人手姿态重定向到机器人手的关节空间。重定向公式在遵守Allegro Hand的关节极限和运动学约束的同时，最小化人手和机器人指尖位置之间的差异。手臂运动通过IK将手腕姿态映射到Kuka IIWA末端执行器来单独控制。

关键创新在于无标记、无手套的设计：操作者只需在相机前移动裸手。这消除了对专用遥操作硬件（VR控制器、触觉手套、电磁追踪器）的需求，降低了成本和设置时间。系统支持手臂系统全部23个驱动自由度，使复杂操作超越简单抓放成为可能。

## 主要贡献

- 首个用于完整灵巧手臂系统（23个驱动自由度）的基于视觉（裸手、无标记/手套）的遥操作系统
- 基于优化的重定向，在运动学可行性约束下将人手关键点映射到Allegro Hand关节空间
- 仅使用单目RGB相机作为输入即展示了复杂操作任务（超越抓放）
- 证明了基于视觉的手部追踪足以用于灵巧遥操作，相比手套/外骨骼系统降低了硬件成本

## 局限性/不足

- 对操作者无力/触觉反馈；操作完全依赖视觉反馈
- 基于视觉的手部追踪对遮挡（手指相互遮挡）和光照条件敏感
- 单相机设置提供有限的深度信息，可能导致重定向歧义
- 视觉处理管线的延迟可能限制高频接触丰富任务
- 仅在Allegro Hand + Kuka IIWA上评估；无跨平台验证
- 无策略学习组件；纯遥操作系统（采集的数据可用于下游模仿学习，但论文中未演示）

## 数据质量影响

DexPilot采集的是仅包含位置的示教数据，无力或触觉信息。基于视觉的手部追踪引入噪声和偶发的追踪失败（尤其在遮挡时），相比手套或外骨骼系统降低了示教质量。然而，低成本和易设置性使收集更大的数据集成为可能，有望通过数据量弥补单次示教质量的不足。系统证明了无需专用硬件即可采集高维灵巧示教数据。

## 定量结果

论文在多个操作任务上评估了两位人类示教者的速度和可靠性指标。任务包括抓取各种物体、手内重新定位和工具操作。具体成功率数值按任务类别报告，但因示教者技能水平而异。

| 指标 | 数值 |
|------|------|
| 控制的总驱动自由度 | 23（16手+7臂） |
| 输入硬件 | 单目RGB相机 |
| 手部追踪 | 基于视觉的关键点检测 |
| 重定向 | 基于优化 |
