### 4.3 HumanoidVLM

**Full title:** VLM-based Impedance Control for Safe Physical Human-Robot Interaction with Humanoid Robots

**Authors:** (Multiple authors, likely overlapping with SafeHumanoid group)

**Venue/Year:** HRI 2026

**How K/D are determined:** VLM (Molmo-7B, open-weight) + FAISS-based RAG retrieval. The system uses a vector database of prior impedance configurations indexed by task context and visual scene features. Given a new interaction scenario, the VLM retrieves similar prior configurations from the FAISS database and adapts them to the current context. Molmo-7B (a 7B-parameter open-weight VLM) performs the reasoning, avoiding dependence on commercial APIs.

**What is output:** Retrieved and adapted Cartesian stiffness K and damping D parameters for the humanoid's end-effector impedance controller. Parameters govern the compliance of the humanoid's arms during physical human-robot interaction.

**Robot platform:** Unitree G1 humanoid robot. No standalone dexterous hand (uses G1's integrated Dex3-1 gripper, 7 DoF per hand).

**Tasks:** Physical human-robot interaction (pHRI) scenarios: collaborative object handover, guided manipulation, physical assistance tasks. Focus is on safety-critical interactions where the humanoid must be compliant to avoid injuring the human partner.

**Key methodology:** The system combines a VLM for semantic understanding of the interaction context with RAG-based retrieval of appropriate impedance parameters. Molmo-7B processes the visual scene (including the human partner's pose and intent cues) and retrieves impedance configurations from a FAISS database that were successful in similar prior interactions. The retrieved K/D values are applied to the humanoid's Cartesian impedance controller, enabling safe and adaptive physical interaction. The use of an open-weight VLM (Molmo-7B) rather than a commercial API distinguishes this from CompliantVLA-adaptor and OmniVIC.

**Architecture/Parameters:** Molmo-7B (open-weight VLM, ~7B parameters) for scene understanding and impedance reasoning. FAISS vector database for nearest-neighbor retrieval of impedance configurations. Cartesian impedance controller on Unitree G1. The pipeline processes camera input, generates scene embeddings, retrieves matching impedance profiles, and applies them to the low-level controller.

**Main contributions:**
- Applies VLM-guided impedance control to humanoid robots for physical human-robot interaction
- Uses open-weight VLM (Molmo-7B) rather than commercial API, enabling on-device deployment
- Demonstrates RAG-based impedance retrieval for safety-critical pHRI scenarios on a full humanoid platform

**Limitations/Gaps:**
- No standalone dexterous hand -- relies on G1's integrated simple gripper
- Impedance parameters are retrieved/adapted at the task level, not continuously modulated during contact
- Limited to the G1 humanoid platform; generalization to other humanoids not demonstrated
- RAG database requires manual population with initial impedance configurations
- Does not address multi-finger contact force distribution
- Safety validation is primarily empirical rather than formally guaranteed
- VLM inference latency (~100ms-1s) limits impedance updates to ~1-10 Hz, three orders of magnitude below the 100-1000 Hz required by real-time impedance controllers.

**Results:** Demonstrated safe physical interactions on the Unitree G1 humanoid, with the VLM-guided impedance controller producing appropriate compliance levels for different interaction types. Reduced peak interaction forces compared to fixed-impedance baselines during handover and collaborative manipulation tasks.

## Inference / Deployment

- **Inference latency:** Not explicitly benchmarked. Molmo-7B inference (open-weight, on-device) likely requires ~200-500ms per query on a modern GPU, plus FAISS retrieval time (<10ms). Impedance parameters are set per task phase.
- **Deployment hardware:** Unitree G1 humanoid. VLM inference (Molmo-7B) can run on a local GPU (specific model not reported), avoiding cloud API dependency. FAISS retrieval runs locally.
- **Real-time capable?** No, for impedance adaptation. The low-level impedance controller on the G1 runs at real-time rates, but VLM-based impedance updates are limited to ~1-10 Hz. The use of an open-weight model (Molmo-7B) enables on-device deployment but does not achieve real-time continuous impedance modulation.

## Dataset / Data Collection

- **Dataset used:** Custom FAISS-based RAG database of impedance configurations for pHRI scenarios. No standard benchmark dataset used.
- **Collection method:** RAG database manually populated with initial impedance configurations from prior pHRI trials on Unitree G1. Real-world experiments with human interaction partners. No teleoperation -- impedance parameters are generated by VLM (Molmo-7B) reasoning, not learned from demonstrations.
- **Data scale:** Not reported. RAG database size and number of evaluation trials not specified in available materials.
- **Teleop equipment:** Not applicable (VLM-based impedance reasoning system, not imitation learning).
- **Data format:** FAISS vector database entries containing scene embeddings and associated impedance parameter profiles.
- **Publicly available?** Not reported. Paper venue is HRI 2026; public release status unknown.
