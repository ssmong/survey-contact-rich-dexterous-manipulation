### 2.6 HumanoidGen

**전체 제목:** HumanoidGen: Data Generation for Bimanual Dexterous Manipulation via LLM Reasoning

**저자:** Zhi Jing, Siyuan Yang, Jicong Ao, Ting Xiao, Yu-Gang Jiang, Chenjia Bai

**학회/연도:** NeurIPS 2025

**arXiv:** [2507.00833](https://arxiv.org/abs/2507.00833) (참고: 이전에 2505.14680으로 인용, 이는 무관한 논문으로 연결됨)

**핸드 하드웨어:** Inspire 정교한 핸드(핸드당 6 DoF, 팔당 7 DoF; 총 행동 공간 26D)를 갖춘 Unitree H1-2 휴머노이드 로봇. Inspire 핸드는 다지 정교한 핸드로, 정교한 조작 섹션 포함을 확인.

**작업:**
- 20개 탁상 조작 작업
- 양손 조율 작업
- 장기 지평 다단계 조작
- 휴머노이드 구현체 맥락에서 정교한 핸드-물체 상호작용을 요구하는 작업

**핵심 방법론:** HumanoidGen은 대규모 언어 모델(LLM)을 작업 플래너로, Monte Carlo Tree Search (MCTS)를 정책 생성에 사용한다. LLM은 복잡한 조작 지시를 하위 작업 시퀀스로 분해하고, MCTS는 고보상 궤적을 찾기 위해 가능한 행동 계획의 공간을 탐색한다. 확산 정책이 저수준 컨트롤러로 기능하여 휴머노이드의 팔과 손에 대한 연속 행동을 생성한다. 언어는 계획 수준에서 작업을 지정하고 분해하는 데 사용되며, LLM은 조작 전략에 대한 상식 추론을 제공한다.

**아키텍처/파라미터:**
- 고수준 플래너: LLM (요약에 구체적 모델 미공개; 작업 분해 및 MCTS 안내에 사용)
- 탐색: 계획 최적화를 위한 Monte Carlo Tree Search
- 저수준 컨트롤러: 연속 행동 생성을 위한 확산 정책
- 대상 플랫폼: Unitree H1_2 휴머노이드
- 모델 및 데이터 가중치가 HuggingFace에 공개

**주요 기여:**
- LLM의 상식 추론과 MCTS의 체계적 탐색을 결합하여 다양한 조작 정책을 생성하는 LLM 안내 MCTS 프레임워크
- 전체 휴머노이드 플랫폼에서 양손 및 장기 지평 조작을 포함하는 20개 탁상 작업에서 시연
- 프레임워크가 동일 작업에 대해 다양한 정책 솔루션을 생성하여, 솔루션 다양성을 통해 강건성 향상

**한계점:**
- 휴머노이드 통합 핸드 전용; 독립 정교한 핸드 플랫폼(Allegro, Shadow, LEAP)은 다루지 않음
- 힘이나 임피던스 출력 없음; 확산 정책은 위치 목표 생성
- LLM 플래너는 작업 분해 수준에서 작동하며, 세밀한 손가락 제어 수준에서는 아님
- MCTS 탐색은 실시간 응용에서 계산적으로 비용이 클 수 있음
- 평가는 주로 탁상 조작; 도구 사용이나 고도로 접촉이 풍부한 작업은 포함하지 않음

## 추론 / 배포

- **추론 지연 시간:** 논문(arXiv 2507.00833)에 보고되지 않음.
- **배포 하드웨어:** Unitree H1_2 휴머노이드 로봇 플랫폼. 훈련/추론 GPU 미명시.
- **실시간 가능 여부:** 미검증. MCTS 기반 계획은 추론 시 계산 오버헤드를 도입할 수 있으나, 구체적 지연 수치 이용 불가.

## 데이터셋 / 데이터 수집

- **사용 데이터셋:** 맞춤 시뮬레이션 생성 데이터셋. HuggingFace에 공개 (`TeleEmbodied/humanoidgen_dataset`).
- **수집 방법:** ManiSkill 플랫폼을 사용한 시뮬레이션 기반 데이터 생성. `record_data`를 활성화한 작업 솔버 스크립트를 실행하여 데이터를 수집하고, 20개 탁상 조작 작업에 대한 시연 궤적을 생성.
- **데이터 규모:** 명시적으로 보고되지 않음. HuggingFace 데이터셋 저장소는 존재하나 구체적 에피소드/궤적 수는 공개 문서에 공개되지 않음.
- **원격 조작 장비:** 해당 없음 (시뮬레이션 생성 데이터). 실세계 배포는 Unitree H1_2 휴머노이드 로봇 사용.
- **데이터 포맷:** pickle 포맷으로 수집 후 `pkl2zarr.py`를 통해 정책 훈련을 위한 zarr 포맷으로 변환.
- **공개 여부:** 예. 데이터셋이 HuggingFace에 (`TeleEmbodied/humanoidgen_dataset`). 모델이 HuggingFace에 (`TeleEmbodied/humanoidgen_model`). 코드: [GitHub](https://github.com/TeleHuman/HumanoidGen). 참고: 장면 생성 및 추가 작업 생성 코드는 아직 공개되지 않음.
