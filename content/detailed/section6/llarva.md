### LLARVA: Vision-Action Instruction Tuning Enhances Robot Learning

**Full title:** LLARVA: Vision-Action Instruction Tuning Enhances Robot Learning

**Authors:** Dantong Niu, Yuvan Sharma, Giscard Biamby, Jerome Quenum, Yutong Bai, Baifeng Shi, Trevor Darrell, Roei Herzig (Berkeley AI Research, UC Berkeley)

**Venue/Year:** arXiv 2024

**Architecture:** Built on LLaVA 1.5 with a Llama 2 7B language model and CLIP ViT-L/14 vision encoder. Both vision and language encoders are frozen; adaptation is done via LoRA adapters on the LLM and a trainable MLP projection layer that aligns visual features to language token dimensionality. Actions are decoded as language tokens (no specialized action head) --- the model outputs both 2D visual trace predictions and action sequences as structured text. The total parameter count is ~7B (dominated by the Llama 2 backbone).

**Action space:** Varies by robot embodiment (joint or end-effector control, absolute or delta). The model handles heterogeneous action spaces across 13 robot types by encoding control mode information in the instruction prompt. Continuous actions are discretized into text tokens. Evaluation includes 7D EEF control (Franka) and joint-space control (RLBench).

**Dex hand support?** ✗ --- Gripper-only evaluation across all 13 robot embodiments.

**Force/impedance output?** ✗ --- Position targets only.

**Key methodology:** LLARVA introduces vision-action instruction tuning, where the VLA is trained to predict auxiliary 2D visual traces (sequences of (x,y) end-effector trajectory coordinates) alongside robot actions. These traces are generated by training a Faster R-CNN detector on end-effector bounding boxes across diverse robots, then extracting center-point trajectories. The instruction prompt encodes robot type, control mode, task description, and proprioceptive history, enabling a single model to handle 13 different robot embodiments. Training is two-stage: (1) vision-action instruction pretraining on 8.5M image-trace pairs from 37 OXE subsets, then (2) fine-tuning on downstream tasks.

**Training data:** Pretraining: 8.5M image-2D visual trace pairs from 37 Open X-Embodiment subsets spanning 13 robot types. Fine-tuning: RLBench (800 demonstrations per task, 12-18 tasks) and real Franka Panda (1,920 episodes for pick/stack/destack tasks).

**Main contributions:**
- Proposed vision-action instruction tuning with auxiliary 2D visual trace prediction as a self-supervised pretraining objective, improving action prediction by providing an intermediate spatial representation that bridges vision and action.
- Demonstrated that structured natural-language prompts encoding robot type, control mode, and proprioceptive history enable a single VLA to generalize across 13 robot embodiments without embodiment-specific heads.
- Showed that instruction pretraining on 8.5M OXE trace pairs provides +17.5% average improvement and visual trace prediction adds +15% on downstream tasks.

**Limitations/Gaps:**
- 2D visual traces lack depth information; authors note that 3D trace representations would improve fine-grained manipulation.
- Single-view 128x128 input images limit spatial precision for delicate tasks.
- Current architecture cannot process multiple views or leverage depth/voxel representations.
- RLBench performance (43.3%) lags behind 3D-based methods like PerAct (55.3%), highlighting the cost of being 2D-only.
- Limited temporal context for long-horizon tasks despite trace-based memory augmentation.
- No dexterous hand or force-aware evaluation.

## Dataset / Data Collection

- **Dataset used:** Open X-Embodiment (37 subsets, 13 robot types) for pretraining. RLBench simulation and real Franka Panda for fine-tuning.
- **Collection method:** OXE data aggregated from diverse sources. Visual traces generated via Faster R-CNN end-effector detection on OXE episodes. RLBench demonstrations from simulation. Real-robot demonstrations via teleoperation.
- **Data scale:** 8.5M image-trace pairs for pretraining. 800 demonstrations per RLBench task. 1,920 episodes for real Franka tasks.
- **Teleop equipment:** Varies by OXE constituent dataset. Real Franka: not specified.
- **Data format:** RLDS (OXE standard) for pretraining.
- **Publicly available?** OXE data is public. Model weights and code release status unclear.
