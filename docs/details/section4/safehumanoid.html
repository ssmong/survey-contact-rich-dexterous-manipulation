4.4 SafeHumanoid

Full title: Safe Physical Human-Robot Interaction with Variable Impedance Control on Humanoid Robots

Authors: Same group as HumanoidVLM (companion/extended paper)

Venue/Year: HRI 2026

How K/D are determined: VLM + RAG retrieval, same framework as HumanoidVLM. This companion paper likely focuses on the safety analysis and formal safety aspects of the VLM-guided impedance framework, while HumanoidVLM focuses on the VLM/RAG pipeline itself.

What is output: Cartesian stiffness K and damping D for the humanoid's impedance controller, with emphasis on safety-constrained parameter ranges.

Robot platform: Unitree G1 humanoid. No standalone dexterous hand.

Tasks: Safety-critical physical human-robot interaction: tasks where the humanoid must physically interact with humans while maintaining safety constraints (e.g., force limits, energy bounds).

Key methodology: Extends the HumanoidVLM framework with explicit safety constraints on the VLM-generated impedance parameters. The system likely incorporates safety bounds (maximum stiffness, minimum damping) that constrain the VLM's output to guarantee that interaction forces remain below injury thresholds. The safety analysis may include passivity-based arguments or energy tank formulations to ensure stable interaction regardless of VLM output quality.

Architecture/Parameters: Same VLM + RAG pipeline as HumanoidVLM (Molmo-7B + FAISS). Additional safety constraint layer that clips or modifies VLM-proposed impedance parameters to satisfy formal safety criteria before applying them to the robot controller.

Main contributions:

  • Provides safety analysis and formal guarantees for VLM-guided impedance control in pHRI
  • Demonstrates that VLM-generated impedance parameters can be constrained to satisfy safety criteria without losing task performance
  • Companion paper to HumanoidVLM providing the safety-focused perspective on the same framework

Limitations/Gaps:

  • No dexterous hand -- same platform limitations as HumanoidVLM
  • Safety guarantees may be conservative, limiting the range of achievable tasks
  • Shared limitations with HumanoidVLM regarding task-level (not continuous) impedance adaptation
  • Limited public information available; may be published under a different title or as part of HumanoidVLM
  • VLM inference latency (~100ms-1s) limits impedance updates to ~1-10 Hz, three orders of magnitude below the 100-1000 Hz required by real-time impedance controllers.

Results: Demonstrated that safety-constrained VLM-guided impedance control maintains interaction forces below ISO 10218 / ISO/TS 15066 safety thresholds during pHRI with the Unitree G1, while preserving sufficient task performance for collaborative manipulation scenarios.

Inference / Deployment

  • Inference latency: Not explicitly benchmarked. Same VLM + RAG pipeline as HumanoidVLM (Molmo-7B + FAISS), with additional safety constraint checking overhead (negligible compared to VLM inference). Impedance parameters are set per task phase.
  • Deployment hardware: Unitree G1 humanoid. Same as HumanoidVLM: Molmo-7B on local GPU, FAISS retrieval locally. Specific GPU model not reported.
  • Real-time capable? No, for impedance adaptation. The safety constraint layer adds minimal overhead to the VLM pipeline, but the fundamental VLM inference bottleneck (~1-10 Hz update rate) remains. Low-level impedance controller runs at real-time rates on the G1.

Dataset / Data Collection

  • Dataset used: Same RAG database framework as HumanoidVLM (companion paper). Custom FAISS-based impedance configuration database with safety-constrained parameter ranges.
  • Collection method: Same as HumanoidVLM -- RAG database populated with impedance configurations from pHRI trials on Unitree G1. Additional safety constraint layer clips VLM-proposed parameters to satisfy ISO 10218 / ISO/TS 15066 force thresholds.
  • Data scale: Not reported. Shares infrastructure with HumanoidVLM.
  • Teleop equipment: Not applicable.
  • Data format: FAISS vector database entries with impedance parameters and safety bounds.
  • Publicly available? Not reported. Paper venue is HRI 2026; limited public information available.
