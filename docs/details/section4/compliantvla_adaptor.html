4.1 CompliantVLA-adaptor

Full title: Compliant Action Generation via VLA Adaptor for Impedance-Controlled Robot Manipulation

Authors: Luigi Penco, Niko Suenderhauf, Fares Abu-Dakka (IIT Genoa / TU Darmstadt)

Venue/Year: arXiv preprint, 2026

How K/D are determined: VLM (GPT-4o-mini) zero-shot prompting. The VLM receives the current scene image and task description, then outputs per-axis Cartesian stiffness and damping values. No training or fine-tuning of the VLM is required -- the system is entirely training-free, relying on the VLM's zero-shot reasoning about appropriate compliance levels for different tasks and contact conditions.

What is output: 6D Cartesian impedance parameters: translational stiffness (Kx, Ky, Kz) and translational damping (Dx, Dy, Dz). These are fed to a Cartesian impedance controller that tracks the position trajectory generated by a frozen VLA (e.g., OpenVLA or pi0).

Robot platform: Franka Emika Panda + parallel-jaw gripper. No dexterous hand.

Tasks: Contact-rich manipulation tasks including: wiping surfaces, plug insertion, drawer opening/closing, object handover with compliance. Evaluated both in LIBERO simulation benchmark and on real Franka hardware.

Key methodology: The system introduces an "adaptor" module that sits between a frozen VLA (which generates position trajectories) and the robot's impedance controller. The VLM reasons about the task semantics and contact requirements from visual input and generates appropriate stiffness/damping profiles without any task-specific training. The position trajectory from the VLA and the impedance parameters from the VLM adaptor are combined at the impedance controller level, enabling compliant behavior while preserving the VLA's task execution capability.

Architecture/Parameters: Two-module pipeline: (1) a frozen pre-trained VLA (OpenVLA 7B or pi0 3.3B) generating end-effector position trajectories, and (2) GPT-4o-mini as the impedance adaptor generating K/D values via API calls. The Cartesian impedance controller runs on the Franka's real-time control interface. No additional neural network training is involved.

Main contributions:

  • Proposes augmenting a frozen VLA with VLM-generated impedance parameters, enabling compliant manipulation without retraining the VLA
  • Demonstrates that VLMs can zero-shot reason about appropriate stiffness/damping for contact-rich tasks from visual and language context alone
  • Provides a modular, training-free approach that can wrap any position-trajectory VLA with compliance capabilities

Limitations/Gaps:

  • No dexterous hand support -- gripper only on Franka Panda
  • K/D values are generated per task segment, not continuously adapted during contact -- essentially open-loop impedance scheduling
  • Relies on commercial API (GPT-4o-mini) with associated latency and cost
  • No formal stability or passivity guarantees for the VLM-generated impedance values
  • Limited task diversity in evaluation; does not address high-force industrial tasks
  • VLM inference latency (~100ms-1s) limits impedance updates to ~1-10 Hz, three orders of magnitude below the 100-1000 Hz required by real-time impedance controllers.

Results: Demonstrated improved success rates on contact-rich tasks in LIBERO compared to position-only VLA baselines. The compliant adaptor reduced contact forces during insertion and handover tasks. Real-robot experiments on Franka Panda confirmed that VLM-generated impedance parameters produce qualitatively appropriate compliance behavior.

Inference / Deployment

  • Inference latency: Not explicitly benchmarked. The VLM adaptor (GPT-4o-mini API call) adds ~100ms-1s per impedance parameter update. The underlying VLA (OpenVLA or pi0) runs at its native speed for trajectory generation. Impedance parameters are generated per task segment, not per control step.
  • Deployment hardware: Franka Emika Panda with its real-time control interface. VLM inference runs via cloud API (GPT-4o-mini); no on-device GPU requirement for the adaptor module. VLA inference hardware not specified.
  • Real-time capable? Partially. The position trajectory from the VLA and the low-level impedance controller run at real-time rates (1 kHz on the Franka). However, VLM-based impedance updates are limited to ~1-10 Hz due to API latency, making continuous impedance adaptation during fast contacts infeasible.

Dataset / Data Collection

  • Dataset used: LIBERO benchmark (6 of 8 tasks) and ManiSkill benchmark (2 of 8 tasks) for simulation evaluation. A custom force-torque-impedance-scenario context dataset is released for the RAG component.
  • Collection method: Simulation rollout for LIBERO/ManiSkill evaluation; real-world demonstrations on Franka Panda for fine-tuning (50 successful trials per task). Visual input from Logitech USB camera (global view) and Intel RealSense D435i (end-effector mounted), images resized to 224x224.
  • Data scale: 50 demonstrations per task for real-world fine-tuning; simulation evaluation across 8 representative tasks.
  • Teleop equipment: Not applicable (VLM-based system is training-free; demonstrations collected for VLA fine-tuning, method not specified).
  • Data format: RGB images + proprioceptive states + language instructions; force-torque-impedance context dataset format not specified.
  • Publicly available? Code, prompts, and force-torque-impedance-scenario context datasets released at https://sites.google.com/view/compliantvla
