Scaffolding+VLM

전체 제목: Scaffolding Dexterous Manipulation with Vision-Language Models

저자: Vincent de Bakker, Joey Hejna, Tyler Ga Wei Lum, Onur Celik, Aleksandar Taranovic, Denis Blessing, Gerhard Neumann, Jeannette Bohg, Dorsa Sadigh (Stanford / KIT)

학회/연도: NeurIPS 2025 (arXiv:2506.19212)

핸드 하드웨어: KUKA 로봇 팔에 장착된 Allegro Hand (16 DoF). 실세계 설정은 16개 관절 명령 차원의 오른손 Allegro 구성을 사용한다.

작업:

  • 시뮬레이션 (8개 작업): 사과 파지, 병 조작, 망치 사용, 서랍 열기/닫기, 냉장고 상호작용, 스펀지 조작, 펜치 작동, 가위 사용.
  • 실세계 (3개 작업): 팔을 사용한 상자 조작 (box_arm), 팔을 사용한 병 조작 (bottle_arm), 팔을 사용한 망치 사용 (hammer_arm).

핵심 방법론: 이 프레임워크는 비전-언어 모델(Gemini 2.5 Flash Thinking)을 사용하여 정교한 조작 시연의 생성을 스캐폴딩하고, 이를 RL 정책 훈련에 사용한다. 파이프라인은 두 단계로 구성된다: (1) VLM 안내 데이터셋 생성, Gemini가 조작 웨이포인트(3, 5, 10, 40개 웨이포인트 실험), 소수 시연, 스크립트 기준선을 생성; (2) VLM 생성 데이터셋에 대한 RL 훈련. 이 접근법은 VLM의 공간 추론을 활용하여 정교한 조작 정책을 부트스트랩하며, 인간 원격 조작 시연의 필요성을 제거한다.

아키텍처/파라미터: VLM 백본으로 Gemini 2.5 Flash Thinking 사용. RL 정책은 VLM 생성 데이터에서 훈련됨. 실세계 배포 스택은 ROS, ZED 카메라, 물체 자세 추정을 위한 FoundationPose, 세그멘테이션을 위한 Segment Anything 2, 키포인트 추적을 통합한다.

시뮬레이션 플랫폼: 맞춤 Gymnasium 환경 (EnvApple-v0, EnvBottle-v0 등). Sim-to-real 전이가 시연됨 -- VLM 스캐폴딩 데이터로 시뮬레이션에서 훈련된 정책이 실제 Allegro + KUKA 시스템에 배포된다.

주요 기여:

  • 정교한 조작을 위한 인간 원격 조작의 대안으로 VLM 스캐폴딩 시연 생성을 도입하며, Gemini의 공간 추론을 활용하여 조작 웨이포인트를 생성한다 -- 이전 정교한 조작 파이프라인은 인간 시연이나 수동 보상 엔지니어링이 필요했다.
  • VLM 생성 시연이 시뮬레이션에서 8개 작업 모음과 실세계에서 3개 작업의 정교한 조작 정책을 성공적으로 훈련할 수 있음을 시연하여, VLM 공간 추론이 정교한 작업 부트스트랩에 충분함을 확립한다.
  • 이 섹션에서 유일하게 VLA/언어 모델을 정교한 조작 파이프라인에 통합한 논문; 다른 모든 연구는 순수 RL이나 인간 시연 파이프라인을 사용한다.

한계점:

  • 힘/임피던스 인식: 없음. 힘/임피던스 제어 사용 없음. VLM은 위치 웨이포인트만 생성; 도구 사용이나 물체 상호작용 중 접촉 힘 프로파일은 명시적으로 스캐폴딩되거나 제어되지 않는다. 정밀한 힘 적용이 필요한 작업(예: 망치 타격 힘, 가위 절단 압력)의 경우, 힘 조절은 하류 RL 정책에 의해 암묵적으로 학습되어야 한다.
  • VLA/언어 조건부: 예 (시연 생성용). Gemini 2.5 Flash Thinking이 스캐폴딩을 위한 VLM 백본으로 사용되나, 실시간 정책 백본이 아닌 오프라인 시연 생성을 수행한다. VLM의 역할은 폐루프 제어가 아닌 데이터셋 생성이다.
  • Sim-to-real: 예. VLM 스캐폴딩 데이터로 시뮬레이션에서 훈련된 정책이 실제 Allegro + KUKA 시스템에 3개 작업으로 배포된다.
  • 코드/데이터 공개: 모델 가중치 공개 없음. 접근법은 스캐폴딩 단계를 위한 Gemini API 접근과 실세계 물체 자세 추정을 위한 FoundationPose에 의존한다.

결과: 8개 시뮬레이션 작업과 3개 실세계 작업에서 평가. 3개 시드(시뮬레이션)와 작업당 20회 롤아웃(실세계)에 걸쳐 성공률 보고.

추론 / 배포

  • 추론 지연 시간: 보고되지 않음. VLM(Gemini 2.5 Flash Thinking)은 추론 시가 아닌 시연 생성을 위해 오프라인으로 사용된다. 하류 RL 정책의 추론 속도는 특성화되지 않았다.
  • 배포 하드웨어: 실세계 시스템은 Allegro Hand (16 DoF) + KUKA 팔, ZED 카메라를 사용하며, 인식을 위해 FoundationPose와 Segment Anything 2를 사용한다. 정책 추론용 GPU 미명시.
  • 실시간 가능 여부: RL 정책은 실시간으로 실행될 가능성이 있으나(표준 MLP 기반 RL 정책은 서브밀리초), 논문에서 제어 주파수나 추론 지연을 보고하지 않는다. VLM 스캐폴딩 단계는 오프라인이며 배포 시 성능에 영향을 주지 않는다.

데이터셋 / 데이터 수집

  • 사용 데이터셋: 맞춤 VLM 생성 시연 (명명된 데이터셋 없음). 시연은 인간이 아닌 Gemini 2.5 Flash Thinking에 의해 합성된다.
  • 수집 방법: VLM 스캐폴딩 합성 생성. Gemini 2.5 Flash Thinking이 조작 웨이포인트(3, 5, 10, 40개 웨이포인트 실험), 소수 시연, 스크립트 기준선을 생성한다. 시연 생성에 인간 원격 조작이 필요 없다.
  • 데이터 규모: 보고되지 않음. 작업당 생성된 시연 수는 공개 자료에 명시되지 않았다.
  • 원격 조작 장비: 해당 없음 (원격 조작이 아닌 VLM 생성 시연).
  • 데이터 포맷: 보고되지 않음.
  • 공개 여부: 데이터셋이나 모델 가중치 공개 없음. 접근법은 스캐폴딩 단계를 위한 Gemini API 접근에 의존한다.
