SimToolReal

전체 제목: From Sim to Real: A Pipeline for Training and Deploying Dexterous Tool-Use Policies

저자: Tyler Lum et al. (Stanford IPRL)

학회/연도: arXiv 프리프린트, 2026

핸드 하드웨어: KUKA 로봇 팔에 장착된 Sharpa Hand (22 DoF). Sharpa Hand는 복잡한 조작 작업에서 다지 조율이 필요한 고DoF 인간형 정교한 핸드이다.

작업: 망치 타격, 스크류드라이버 회전, 주걱 뒤집기 등 24가지 도구 사용 작업 평가. 작업 모음은 조율된 손가락-도구-물체 상호작용이 필요한 광범위한 일상 도구 사용 행동을 포괄하도록 설계되었다.

핵심 방법론: SimToolReal은 시뮬레이션에서 정교한 도구 사용 정책을 훈련하고 실제 하드웨어에 배포하기 위한 엔드-투-엔드 파이프라인을 제안한다. 도메인 랜덤화를 포함한 IsaacGym에서의 강화학습 정책 훈련으로 강건한 sim-to-real 전이를 달성한다. 파이프라인은 시뮬레이션 환경 설계부터 정책 훈련, 22-DoF Sharpa Hand에서의 실세계 배포까지 전체 워크플로를 다룬다.

아키텍처/파라미터: IsaacGym 대규모 병렬 환경에서 PPO로 훈련된 RL 정책. 정책 네트워크는 일반적으로 고유감각 상태 입력을 사용하는 MLP 기반. 체크포인트가 공개적으로 배포되었다.

시뮬레이션 플랫폼: NVIDIA IsaacGym. Sim-to-real 전이가 시연됨 -- 전적으로 시뮬레이션에서 훈련된 정책이 미세 조정 없이 실제 하드웨어에 배포된다.

주요 기여:

  • 24개 작업 규모의 정교한 도구 사용을 위한 완전한 sim-to-real 파이프라인을 최초로 시연하며, 이전 연구는 고DoF 핸드로 기껏해야 소수의 도구 사용 작업을 다루었다.
  • 다양한 도구 범주(타격, 회전, 뜨기 등)를 포괄하는 가장 큰 정교한 도구 사용 평가 모음을 제공하며, 이전 벤치마크는 단일 도구 유형에 초점을 맞추었다.
  • 코드(GitHub)와 모델 체크포인트를 오픈 소스로 공개하여 재현성을 보장한다 -- 대부분의 비교 연구는 훈련된 정책을 공개하지 않는다.

한계점:

  • 힘/임피던스 인식: 없음. 시스템은 힘/토크 센싱이나 임피던스 제어를 포함하지 않으며, 순수 위치 기반 제어 목표에 의존한다. 상당한 접촉 힘이 수반되는 작업(해머링, 지렛대 사용)에서 접촉 관리는 명시적 힘 조절이 아닌 시뮬레이션 접촉 모델과 하드웨어 유연성을 통해 암묵적으로 처리된다.
  • VLA/언어 조건부: 없음. 정책은 언어나 비전-언어 모델 통합 없는 작업별 RL 에이전트이다.
  • Sim-to-real: 예. IsaacGym에서 전적으로 훈련된 정책이 도메인 랜덤화를 사용하여 sim-to-real 격차를 극복하며, 미세 조정 없이 실제 하드웨어에 배포된다.
  • 코드/데이터 공개: 코드 공개 (GitHub); 모델 체크포인트 공개.

결과: 24개 작업, 12개 물체 인스턴스, 6개 도구 범주(망치, 마커, 지우개, 브러시, 주걱, 스크류드라이버)에 걸쳐 120회 실세계 롤아웃에서 평가. 논문은 이진 성공률이 아닌 작업 진행률(epsilon=2cm 허용 범위 내 추적된 시연 목표 자세의 백분율)을 보고한다.

기준선 대비 개선 +37% 작업 진행률에서 SimToolReal vs. 이전 리타게팅 및 고정 파지 방법
실세계 롤아웃 120 24개 작업, 12개 물체 인스턴스, 6개 도구 범주에 걸쳐
최고 성능 범주 지우개 작업 병진 집중 궤적 (최고 작업 진행률)
전문가 비교 동등 SimToolReal이 훈련 조건에서 전문가 정책과 동등하면서 궤적 변동에 대한 강건성 유지

도구 범주별 결과(논문 그림에서): 지우개와 마커 작업이 가장 높은 작업 진행률 달성; 망치, 브러시, 주걱, 스크류드라이버 작업(인핸드 회전 필요)은 중간이나 성공적 성능. 운동학적 리타게팅 기준선은 대부분의 경우 파지에 실패. 고정 파지 기준선은 회전 필요 작업에서 성능 저하.

추론 / 배포

  • 추론 지연 시간: 명시적으로 보고되지 않음. 논문은 실세계 추론 중 30 Hz 물체 자세 추적과 훈련 중 관측 지연 및 행동 실행 지연에 대한 도메인 랜덤화를 언급하나, 단계별 정책 추론 시간을 정량화하지 않는다.
  • 배포 하드웨어: 실세계 배포를 위한 KUKA 팔 + Sharpa Hand (22 DoF) + ZED 스테레오 카메라. 정책 추론용 GPU 모델 미보고.
  • 실시간 가능 여부: RL 기반 MLP 정책의 경우 가능할 것으로 보이나(일반적으로 서브밀리초 추론), 구체적 제어 주파수 미보고. 30 Hz 자세 추정은 제어 루프가 30 Hz 이하로 실행됨을 시사한다.

데이터셋 / 데이터 수집

  • 사용 데이터셋: DexToolBench (본 연구를 위해 생성된 맞춤 벤치마크), 6개 도구 범주 포괄: 망치, 마커, 지우개, 브러시, 주걱, 스크류드라이버, 범주당 여러 물체 인스턴스와 작업 변형 포함.
  • 수집 방법: IsaacGym에서의 RL 시뮬레이션 롤아웃. ZED 카메라로 캡처된 실세계 RGB-D 시연(SAM 2, SAM 3D를 통한 메시 추출, FoundationPose를 통한 6D 자세 추정으로 처리)이 작업 궤적 정의에 사용되며, 시뮬레이션 훈련을 위한 JSON 자세 파일로 변환된다.
  • 데이터 규모: 명시적으로 보고되지 않음 (물체 범주/물체/작업별로 계층적 구조).
  • 원격 조작 장비: 해당 없음 (시뮬레이션에서 RL 훈련; 실세계 시연은 원격 조작이 아닌 수동 RGB-D 녹화로 캡처).
  • 데이터 포맷: RGB 이미지, 깊이 이미지, 물체 마스크, 카메라 내부 파라미터(cam_K.txt), 로봇 프레임 내 자세(poses.json), 작업별 디렉토리에 구성.
  • 공개 여부: 예. download_dextoolbench_data.py로 벤치마크 데이터 다운로드 가능; download_pretrained_policy.py로 사전 훈련된 정책 체크포인트 다운로드 가능 (GitHub).
