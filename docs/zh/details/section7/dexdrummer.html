7.4 DexDrummer

完整标题： DexDrummer: Dexterous Drumming via Reinforcement Learning

作者： Hao-Chun Fang, Kenneth Shaw, Deepak Pathak等（Stanford）

发表venue/年份： arXiv 2026

arXiv： https://arxiv.org/abs/2603.22263

乐器演奏差异化： 以工具抓取（鼓棒）和冲击力控制区分。与钢琴演奏（RoboPianist、HandelBot）中手指直接接触琴键不同，打鼓需要握持工具并在鼓面上产生控制的冲击力——结合了抓取稳定性和动态击打。仅仿真。

RL算法： PPO + 乐谱条件奖励。与HandelBot类似框架但适配打击乐。需要学习在鼓面上产生冲击力。

手部硬件： 双手灵巧手（握持鼓棒）

仿真平台： IsaacGym / MuJoCo

仿真到真实？ 否（仿真结果；初始预印本未报告真实世界迁移）。

任务： 打鼓——根据乐谱在正确时间以正确强度击打鼓面。涉及握持鼓棒（工具使用）、快速接触丰富冲击、双手协调和动态力控。

核心方法论： DexDrummer将乐器演奏扩展到打击乐，引入超越钢琴的额外挑战：智能体必须握持工具（鼓棒）、产生控制的冲击力（不仅是按键）、协调快速双手交替击打。鼓棒击鼓的接触动力学比钢琴按键更剧烈和动态，需要仿真中的鲁棒接触处理。

主要贡献：

  • 将灵巧乐器操作扩展到打击/打鼓，需要工具使用和冲击力控制
  • 展示RL可学习使用握持工具的动态击打运动，结合抓取稳定性和冲击生成
  • 解决交替和同时击打的双手协调

局限性/差距： 未展示真实世界迁移。快速击打期间的鼓棒抓取稳定性是显著的仿真到真实挑战。评估中的节奏复杂度有限。

仅仿真限制： 仅在仿真中评估；未展示仿真到真实迁移，真实世界可行性未验证。

结果： 在仿真中实现了正确节拍和力度模式的成功打鼓（仿真）。代码公开可用。

数据集 / 数据收集

  • 使用的数据集： 作为任务规格的打击乐谱。无预收集机器人演示数据集。
  • 收集方法： IsaacGym/MuJoCo中的纯RL（PPO）+ 乐谱条件奖励。双手灵巧手握持鼓棒。智能体从奖励反馈中学习工具抓取、冲击力生成和节拍。仅仿真。
  • 数据规模： 不同难度的乐谱。标准并行RL训练。
  • 遥操作设备： 不适用（纯RL，无演示）。
  • 数据格式： 乐谱（带节拍的打击音符序列）。
  • 是否公开？ 代码公开可用。

推理 / 部署

  • 推理延迟： 未报告。MLP策略每次前向传播 <1ms。
  • 部署硬件： 仅仿真（IsaacGym/MuJoCo）。未展示真实机器人部署。
  • 支持实时？ 是，就策略本身而言（MLP推理速度极快）。但仅进行了仿真评估。
