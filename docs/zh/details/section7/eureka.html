7.4 Eureka

完整标题： Eureka: Human-Level Reward Design via Coding Large Language Models

作者： Yecheng Jason Ma, William Liang, Guanzhi Wang, De-An Huang, Osbert Bastani, Dinesh Jayaraman, Yuke Zhu, Linxi Fan, Anima Anandkumar

发表venue/年份： ICLR 2024

arXiv： https://arxiv.org/abs/2310.12931

RL算法： PPO（IsaacGym RL任务）；LLM生成奖励函数代码，用于训练PPO策略。基于LLM变异的奖励候选进化优化。

手部硬件： 仿真中Shadow Hand（24自由度）（转笔任务）；另外9种机器人形态跨29个环境

仿真平台： IsaacGym（NVIDIA GPU加速RL环境）

仿真到真实？ 否。所有结果在仿真中。论文聚焦于奖励设计，非仿真到真实迁移。

任务： 跨10种机器人形态的29个任务。旗舰任务：Shadow Hand转笔（该任务首个成功的RL解）。其他任务包括倒立摆、四足运动、人形和各种操作基准。

核心方法论： 使用GPT-4从环境源代码和任务描述生成奖励函数代码。进化搜索每轮提出多个奖励候选，为每个训练RL策略，评估适应度，将结果反馈给LLM进行改进。LLM使用环境代码（非自然语言任务描述）作为上下文，使其能编写可执行奖励函数。可选地通过自然语言融入人类反馈，通过上下文学习改进奖励。

主要贡献：

  • 提出通过LLM代码生成的自动奖励设计，在IsaacGym基准上跨多样RL任务超越手动设计的奖励函数（83%任务，52%平均归一化提升）
  • 首个成功的仿真中Shadow Hand转笔RL策略（无真实世界迁移），通过Eureka生成奖励结合课程学习实现
  • 展示LLM可从环境源代码生成奖励函数，无需领域特定提示
  • 用于融入人类反馈的无梯度上下文学习方法

定量结果：

指标 数值
超越人类奖励的任务 83%（29任务，10种机器人形态）
平均归一化提升 52%超过专家人类奖励
转笔 首个成功的RL解（Shadow Hand）
使用的LLM GPT-4
每轮奖励候选 多个（进化搜索）

局限性/差距：

  • 力控： 无力/扭矩传感或阻抗控制；所有任务使用位置控制
  • VLA/语言： LLM用于奖励生成（非策略）；无语言条件策略
  • 仿真到真实： 无真实世界实验；所有结果在仿真中
  • 代码： GitHub 可用
  • 计算： 需要大量GPU计算用于并行奖励候选评估和显著LLM API成本
  • 泛化： 奖励函数是任务特定的；无跨任务的奖励设计迁移
  • 所有结果仅在仿真中；"超人"比较是与IsaacGym中手动设计的奖励函数对比，非与人类物理表现对比。

推理 / 部署

  • 推理延迟： 未报告。训练后的MLP策略每次前向传播 <1ms。LLM奖励生成为离线（非部署部分）。
  • 部署硬件： 仅仿真（IsaacGym）。无真实机器人部署。LLM（GPT-4）仅离线用于奖励函数生成。
  • 支持实时？ 是，就训练后策略而言（MLP推理速度极快）。Eureka的贡献在于奖励生成，非推理优化。
