# 10.3 AnyTeleop

  • 全称： AnyTeleop: A General Vision-Based Dexterous Robot Hand-Arm Teleoperation System
  • 作者： Yuzhe Qin, Wei Yang, Binghao Huang, Karl Van Wyk, Hao Su, Xiaolong Wang, Dieter Fox (UCSD, NVIDIA)
  • 发表venue/年份： RSS 2023
  • 输入模态： 单目RGB相机（基于视觉的手部姿态估计）
  • 目标手： 多种灵巧手（Allegro、Shadow等，通过模块化重定向支持）
  • 力反馈： 无
  • 成本： 极低（仅需网络摄像头）

核心方法/设计

AnyTeleop使用单目RGB相机的基于视觉的手部姿态估计来驱动灵巧机器人手。系统将手部姿态估计与机器人特定的重定向解耦，使用模块化重定向模块将检测到的人手姿态映射到任意支持的机器人手。这种设计使遥操作能够跨越不同手平台，无需硬件特定的传感器或手套。纯视觉方案将设置成本和复杂度降至最低。

主要贡献

  • 纯相机遥操作，无需可穿戴传感器、手套或VR硬件
  • 模块化重定向架构，通过单一输入管线支持多种机器人手平台
  • 灵巧遥操作的最低成本入口（仅需网络摄像头）

局限性/不足

  • 基于视觉的手部追踪精度低于传感器方案，尤其在手指遮挡或快速运动时
  • 无力反馈，使操作者难以完成接触丰富任务
  • 单相机设置限制了深度感知，可能引入追踪抖动
  • 视觉处理管线的延迟可能影响实时控制质量

数据质量影响

示教数据缺乏力信息，且位置精度低于传感器方案，产生力调节能力差的策略。单相机追踪在手指快速运动或遮挡时引入抖动和偶发的追踪丢失，表现为示教轨迹中的噪声。基于此数据训练的策略同时继承了力盲和位置噪声，导致手指放置不精确且无法学习接触力调节。对于需要受控接触的任务（如插入、枢转、表面跟随），AnyTeleop的示教数据不足以学习所需的力-位置协调。
