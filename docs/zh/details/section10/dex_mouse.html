# 10.6 DEX-Mouse

  • 全称： DEX-Mouse: A Handheld Interface for Dexterous Teleoperation with Universal Applicability
  • 作者： 未在综述中指明
  • 发表venue/年份： arXiv 2604.15013, 2026年4月
  • 输入模态： 手持物理接口（类鼠标设备）
  • 目标手： 通用（任意灵巧手）
  • 力反馈： 通过设备力学提供运动觉反馈
  • 成本： <$150

核心方法/设计

DEX-Mouse将灵巧遥操作输入重新构想为手持设备，而非穿戴式手套或外骨骼。操作者操控一个紧凑的物理接口——外形类似鼠标——将手部运动映射为灵巧机器人手命令。设备通过其机械结构提供运动觉反馈。低于150美元的价格使其成为本综述中成本最低的遥操作接口，大幅降低了灵巧数据采集的门槛。

主要贡献

  • 本综述中成本最低的灵巧遥操作接口（<$150），大幅降低灵巧数据采集壁垒
  • 手持形态消除了可穿戴设备的标定和穿脱开销
  • 跨不同机器人手平台的通用适用性
  • 通过设备力学提供运动觉反馈

局限性/不足

  • 手持接口无法以与手套方案相同的保真度捕捉独立手指运动
  • 运动觉反馈仅限于设备级力，非逐指接触渲染
  • 从设备运动到高自由度手命令的映射可能需要非直觉的操作者训练
  • 非常新（2026年）；社区验证有限

数据质量影响

手持形态从根本上限制了采集示教数据的维度：单个设备无法同时独立控制所有15-20+个手指关节，因此从设备运动到手关节命令的映射引入了耦合并降低了采集轨迹的表达能力。基于DEX-Mouse数据训练的策略可能能有效学习较粗糙的抓取策略（如力量抓取），但缺乏精密操作所需的独立手指控制（如手内重定向、转笔）。通过设备力学提供的运动觉反馈提供了一定的接触感知，但仅在设备层面而非逐指层面，限制了下游策略中的力调制行为。

开源状态

已开源（硬件+软件）。
