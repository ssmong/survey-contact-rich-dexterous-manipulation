# 10.5 DEXOP

  • 全称： DEXOP: Dexterous Teleoperation with a Passive Exoskeleton
  • 作者： Stanford团队
  • 发表venue/年份： arXiv 2509.04441, 2025年9月
  • 输入模态： 佩戴在操作者手上的被动外骨骼
  • 目标手： 任意灵巧手（通用重定向）
  • 力反馈： 通过外骨骼机械耦合的本体感觉反馈
  • 成本： 未报告（研究原型）

核心方法/设计

DEXOP使用操作者佩戴在手上的被动（无动力）外骨骼。外骨骼在机械上镜像目标机器人手的运动学，因此操作者的手指运动通过运动学耦合直接驱动机器人手关节。由于外骨骼是被动的，它提供固有的本体感觉反馈——操作者可以感受外骨骼结构的机械阻力，近似模拟机器人手的关节极限和机械阻抗。这种方法无需电子传感器或执行器即可实现基本力反馈。

主要贡献

  • 被动（无动力）外骨骼设计，无需电子器件即可提供固有本体感觉反馈
  • 通过模块化运动学耦合实现对任意灵巧手平台的通用重定向
  • 机械简洁性确保了稳健、低维护的运行

局限性/不足

  • 本体感觉反馈反映的是外骨骼力学，而非实际的机器人-环境接触力
  • 外骨骼设计必须针对不同机器人手的运动学进行定制
  • 无主动力渲染，系统仅限于被动顺应性反馈
  • 笨重的外形可能在长时间数据采集中导致操作者疲劳

数据质量影响

被动外骨骼提供的本体感觉反馈帮助操作者遵守关节极限并避免运动学不可行的构型，相比纯视觉系统提高了位置精度。然而，反馈反映的是外骨骼力学而非实际的机器人-环境接触力。示教数据捕捉运动学上合理的轨迹，但缺乏来自操作任务本身的力信息。基于DEXOP数据训练的策略受益于更干净的关节空间轨迹（更少的重定向伪影），但对于接触丰富任务仍缺乏力感知。本体感觉信号有助于提升粗运动质量，但不能解决限制下游策略在需要力调节任务上性能的根本性力盲问题。
