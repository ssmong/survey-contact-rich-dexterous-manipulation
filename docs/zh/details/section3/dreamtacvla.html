3.6 DreamTacVLA

  • 全称: Learning to Feel the Future: Tactile World Model for Contact-Rich VLA Policy
  • 作者: 西北大学等
  • 发表平台/年份: arXiv预印本，2025
  • arXiv: https://arxiv.org/abs/2512.23864

力/触觉输入类型： 视觉触觉传感器。触觉图像通过V-JEPA2（自监督视觉编码器）处理以提取触觉潜在表示，而非直接使用原始触觉图像。

力/阻抗输出： 否。仅位置动作输出。但模型内部预测未来的触觉潜在状态（触觉世界模型），用于优化动作预测。

局限性（仅位置输出）： 仅位置输出；尽管接收了力/触觉输入，策略无法主动调节接触力。

机器人平台： Dobot Xtrainer机械臂 + 夹爪 + 视觉触觉传感器。

局限性（仅夹爪）： 仅在平行夹爪上进行了评估；未在多指灵巧手上验证。

任务： 4项需要触觉反馈才能成功的接触丰富操作任务。这些任务旨在评估预测未来触觉状态是否能改善操作性能。

关键方法： DreamTacVLA引入了一个触觉世界模型，基于当前状态和计划的动作预测未来的触觉潜在表示。V-JEPA2编码器将原始触觉图像处理为紧凑的潜在向量。世界模型模块（基于Transformer）学习预测给定候选动作序列下触觉潜在状态的演变。这个预测的未来触觉状态随后用于优化动作预测——有效地让策略"感受未来"并避免会导致不良接触状态（如过度力、滑动）的动作。

架构/参数： VLA主干网络 + V-JEPA2触觉编码器 + 基于Transformer的触觉世界模型。V-JEPA2编码器在大规模视觉数据上预训练，并在触觉图像上微调。世界模型预测未来的触觉潜在状态，反馈给动作解码器进行优化。

主要贡献：

  • 引入触觉世界模型来预测未来的接触状态，实现主动（而非反应式）接触丰富操作。
  • 使用V-JEPA2作为触觉编码器，证明预训练的视觉自监督学习模型能有效编码触觉图像信息。
  • 展示预测未来触觉状态并据此条件化动作，相比触觉无关和仅反应式触觉方法都能提高接触丰富任务的成功率。

局限性： 未发布模型权重（仅GitHub仓库代码）。触觉世界模型增加了推理的计算开销。仅限4项任务，在单一机器人平台上评估。V-JEPA2编码器为自然图像设计，可能无法最优编码触觉信号。该方法假设使用视觉触觉传感器（如GelSight类），不泛化到其他触觉传感器模态。

结果：

指标 数值
最佳成功率 4项接触丰富任务高达95%
相对仅反应式触觉的提升 10--20%

推理/部署

  • 推理延迟： 控制循环以50 Hz运行（与数据收集频率一致）。Think-Dream-Act流水线增加了推理开销；作者承认这一点并建议"策略蒸馏和自适应想象以实现更快的单程推理"作为未来工作。
  • 部署硬件： 在单块NVIDIA GPU上训练（型号未指定；20K步约需8-12小时）。Dobot Xtrainer机械臂 + 夹爪 + 视觉触觉传感器。推理GPU未单独指定。
  • 是否支持实时？ 部分支持。50 Hz控制频率得以维持，但两阶段流水线（世界模型预测 + 动作生成）引入了计算开销。作者明确将推理速度标记为一个局限性。

数据集/数据收集

  • 使用的数据集： 结合高保真数字孪生仿真数据和真实世界实验数据的混合大规模数据集。构建目的是缓解触觉数据稀缺和触觉传感器易磨损的问题。
  • 收集方法： 混合方法：（1）数字孪生仿真生成合成触觉数据，和（2）使用视觉触觉传感器（GelSight类）的真实世界实验。数据以50 Hz收集。
  • 数据规模： 未明确报告。
  • 遥操作设备： 未报告。在Dobot Xtrainer机械臂上使用夹爪和视觉触觉传感器收集数据。
  • 数据格式： 未报告。
  • 是否公开可用？ 代码可用（引用GitHub仓库）。未发布模型权重。数据集可用性未确认。
