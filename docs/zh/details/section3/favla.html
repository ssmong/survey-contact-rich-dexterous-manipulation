3.4 FAVLA

  • 全称: FAVLA: Force-Aware Vision-Language-Action Model for Contact-Rich Bimanual Manipulation
  • 作者: 中国科学技术大学（USTC）等
  • 发表平台/年份: arXiv预印本，2026
  • arXiv: https://arxiv.org/abs/2602.23648
架构说明： 架构上与ForceVLA类似（F/T分词化、交叉注意力、位置输出）；主要创新在于双臂扩展。

力/触觉输入类型： 安装在双臂系统两个机械臂上的六轴F/T传感器（高频）。

力/阻抗输出： 否。两个机械臂均为仅位置动作输出。

局限性（仅位置输出）： 仅位置输出；尽管接收了力/触觉输入，策略无法主动调节接触力。

机器人平台： Monte双臂系统，配备X-ARM机械臂。

局限性（仅夹爪）： 仅在平行夹爪上进行了评估；未在多指灵巧手上验证。

任务： 4项接触丰富的双臂操作任务：USB插入、齿轮啮合、表面擦拭和连接器插入。这些任务需要两个机械臂之间的协调力控制。

关键方法： FAVLA将力感知VLA范式扩展到双臂场景。来自两个机械臂的高频F/T信号通过各自的时序编码器处理，然后通过交叉注意力与视觉token融合。模型生成协调的双臂动作块，同时考虑两个末端执行器的接触状态。力感知注意力机制允许模型根据另一个机械臂的力读数来条件化一个机械臂的动作，实现反应式双臂协调。

架构/参数： 具有双力编码器的双臂VLA。动作空间覆盖两个机械臂（14D：2 x 6自由度 + 2个夹爪）。来自两个机械臂的力输入独立编码，通过交叉注意力层融合。

主要贡献：

  • 将力感知VLA扩展到双臂操作，其中跨两个机械臂的协调力感知至关重要。
  • 证明跨臂力注意力（根据另一个臂的力来条件化一个臂的动作）改善了双臂插入等任务的协调。
  • 在需要两个机械臂同时进行力调节的任务上进行了评估（例如一个臂持物、另一个臂插入）。

局限性： 未发布代码或权重。尽管有力输入但仅位置输出——系统无法主动调节接触力。仅限4项任务，在单一双臂平台上评估。双臂力融合增加了复杂性，但论文未全面消融与更简单融合策略的对比。

结果： FAVLA在所有4项任务上优于单臂力感知VLA和纯视觉双臂基线。USB插入和齿轮啮合任务从双臂力感知中获益最大。

推理/部署

  • 推理延迟： 未明确报告。VLM以"固定低频率"运行，而动作专家通过基于预测力方差的自适应频率调度以"可变高频率"运行。未披露具体的Hz/ms数据。
  • 部署硬件： 在NVIDIA A100 80GB GPU上训练。Monte双臂系统配备X-ARM机械臂用于真实世界部署。推理硬件未单独指定。
  • 是否支持实时？ 未验证。自适应频率机制根据预测的力波动性将执行从1步扩展到N_max步，但未提供具体延迟数据。

数据集/数据收集

  • 使用的数据集： 4项双臂接触丰富任务的自定义演示数据集。无命名基准。
  • 收集方法： 在Monte双臂系统（X-ARM机械臂）上进行遥操作演示，同步记录来自两个机械臂的高频六轴F/T数据。
  • 数据规模： 未报告。
  • 遥操作设备： 未报告。在Monte双臂系统上使用两个机械臂的F/T传感器进行遥操作。
  • 数据格式： 未报告。
  • 是否公开可用？ 否。未发布代码、数据或权重。
