4.2 OmniVIC

完整标题： OmniVIC: Omnidirectional Variable Impedance Control via Vision-Language Model with Self-Improving RAG

作者： Luigi Penco, Niko Suenderhauf, Fares Abu-Dakka 等 (IIT Genoa / Georgia Tech)

发表venue/年份： arXiv预印本，2025

K/D如何确定： VLM (GPT-4o-mini) + 自改进RAG（检索增强生成）。系统维护一个先前任务执行及其关联阻抗参数和结果的数据库。当新任务呈现时，VLM通过RAG检索相关的先前经验，推理适当的K/D值，并提出阻抗参数。执行后，成功/失败反馈被存回数据库，实现随时间自我改进。

输出内容： 用于全向阻抗控制的完整6D笛卡尔刚度矩阵K和阻尼矩阵D。参数具有方向感知能力，允许在不同笛卡尔轴上设置不同的柔顺性。

机器人平台： Franka Emika Panda + 6轴力/力矩传感器 + 平行夹爪。无灵巧手。

任务： 接触丰富的操作，包括：带力调节的表面擦拭、轴孔插入、物体抛光、柔顺交接。任务要求方向相关的柔顺性（例如，沿插入轴刚性，横向柔顺）。

核心方法： OmniVIC通过自改进的检索增强生成循环扩展了基础VLM阻抗生成。VLM接收任务描述、当前视觉观察以及检索到的类似历史任务及其阻抗参数和结果的示例。然后生成方向特定的K/D值。力/力矩传感器提供力反馈，用于评估执行质量并更新RAG数据库。经过反复试验，系统在无需基于梯度训练的情况下改进其对重复任务类型的阻抗参数选择。

架构/参数： GPT-4o-mini作为VLM主干，FAISS或类似向量数据库用于RAG检索，6轴力/力矩传感器用于执行监控。RAG数据库存储任务描述、视觉特征、使用的阻抗参数和成功指标。阻抗控制器作为Franka上的笛卡尔阻抗控制器运行。

主要贡献：

  • 引入用于阻抗参数选择的自改进RAG，使系统能够在无需梯度更新的情况下从执行结果中学习
  • 展示了从VLM推理生成全向（轴特定）阻抗的能力，超越了标量刚度
  • 表明RAG检索在重复试验中提高了K/D选择精度，优于纯零样本VLM提示

局限性/差距：

  • 无灵巧手——仅在Franka Panda上使用夹爪
  • 改进循环需要多次真实世界试验；样本效率受限于VLM的推理而非从数据中学习
  • 依赖商业VLM API（GPT-4o-mini）
  • RAG数据库需要通过初始试验来填充；冷启动性能等同于零样本VLM
  • 未涉及接触过程中的连续阻抗自适应（参数按任务阶段设定）
  • VLM推理延迟（约100ms-1s）将阻抗更新限制在约1-10 Hz，比实时阻抗控制器所需的100-1000 Hz低三个数量级。

结果： 自改进RAG在所有评估任务中均优于纯零样本VLM阻抗生成。经过多次迭代后，系统收敛到与专家调参值相当的阻抗参数。力/力矩传感器反馈确认，相比固定阻抗基线，峰值力降低且任务成功率提高。

推理 / 部署

  • 推理延迟： 未明确进行基准测试。通过GPT-4o-mini API的VLM推理每次阻抗参数查询增加约100ms-1s，加上RAG检索开销（FAISS最近邻搜索，通常<10ms）。阻抗参数按任务阶段生成，而非按控制步生成。
  • 部署硬件： Franka Emika Panda配备6轴力/力矩传感器。VLM推理通过云API（GPT-4o-mini）运行；FAISS检索在本地机器上运行（硬件未指定）。
  • 是否支持实时？ 阻抗自适应不支持实时。底层阻抗控制器以实时速率运行在Franka上，但VLM阻抗更新因API调用延迟限制在约1-10 Hz。不支持快速接触转换过程中的连续阻抗调制。

数据集 / 数据采集

  • 使用的数据集： LIBERO基准测试（20个任务：LIBERO-Object 10个，LIBERO-Goal 10个）用于仿真评估。从执行经验构建的自定义RAG知识库数据集。
  • 采集方法： 通过VLM辅助的机器人策略执行进行自动采集。基于最大接触力（Fmax）和时间预算（Tmax）阈值标记成功/失败。RAG数据库存储任务指令（原始文本 + BGE-M3嵌入）、末端执行器扭转、重力补偿后力矩、阶段标签和控制器参数（K, D）。在Franka Emika Panda上使用ATI Mini45力/力矩传感器进行真实世界验证。
  • 数据规模： RAG数据库：200条记录（10个任务中每个任务-阶段对20条记录；最大容量20,000）。评估：每种方法每个任务50个episode，覆盖10个查询集任务。三个真实世界接触丰富任务。
  • 遥操作设备： 不适用（自动策略执行，非遥操作）。
  • 数据格式： 包含文本嵌入、力矩数据、阶段标签{Free_motion, Approaching, Contact, Retreat}和阻抗参数的RAG记录。
  • 是否公开可用？ 是——代码、视频和RAG数据集发布于 https://sites.google.com/view/omni-vic
